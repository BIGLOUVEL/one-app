# ONE — Design Principles & Vision

## Core Philosophy
ONE is a **Focus Operating System**, not a task manager. Every design decision must reinforce the "ONE Thing" philosophy.

---

## UI Themes

### Modern (Default)
- **Fond** : Noir profond (`hsl(220, 20%, 4%)`)
- **Accent** : Vert électrique (`hsl(150, 100%, 42%)`)
- **Vibe** : Tech-forward, sleek, focused, immersive
- **Usage** : Landing page, app par défaut

### Elegant
- **Fond** : Papier chaud (`hsl(42, 33%, 96%)`)
- **Accent** : Terracotta (`hsl(15, 70%, 50%)`)
- **Vibe** : Swiss editorial, minimal, warm, sophisticated
- **Usage** : Option utilisateur dans Settings

---

## Design Rules

1. **Less is more** — Chaque élément doit servir un but
2. **Focus first** — L'UI doit éliminer les distractions, pas en créer
3. **Consistency** — Un thème choisi doit être cohérent partout
4. **Motion with purpose** — Les animations renforcent le feedback, pas le spectacle

---

## Red Flags (Quand refuser une demande)

- Mélanger les deux thèmes dans une même vue
- Ajouter des éléments décoratifs sans fonction
- Complexifier l'UI sans gain utilisateur clair
- Affaiblir le contraste ou la lisibilité
- Diluer le message "ONE Thing" avec trop d'options

---

## Process

Quand une demande semble problématique :
1. Identifier pourquoi ça affaiblit la vision
2. Expliquer clairement au user
3. Proposer une alternative cohérente
4. Consulter en cas de doute
